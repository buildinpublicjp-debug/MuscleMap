# /session — 開発セッションサマリー → Obsidian 書き出し

## 概要
現在の開発セッションの成果を集計し、Obsidian デイリーノートに追記する。
デイリーノートが未作成なら新規作成する。

## 手順

### 1. セッションデータ収集
以下を自動で取得:

```bash
# 今日のコミット一覧（ハッシュ + メッセージ）
git log --oneline --since="today 00:00"

# コミット数
git log --oneline --since="today 00:00" | wc -l

# 変更ファイル統計
git diff --stat $(git log --since="today 00:00" --format=%H | tail -1)^..HEAD

# 変更行数
git diff --shortstat $(git log --since="today 00:00" --format=%H | tail -1)^..HEAD
```

### 2. コミット分類
コミットメッセージのプレフィックスで分類:
- `fix:` → バグ修正
- `feat:` → 新機能
- `docs:` → ドキュメント
- `refactor:` → リファクタ
- その他 → 改善

### 3. デイリーノート確認
- パス: `/Users/og3939397/Documents/Obsidian Vault/zDOG/Daily/{YYYY-MM-DD}.md`
- 存在する場合: 既存の内容を読み込む
- 存在しない場合: /daily スキルと同じテンプレートで新規作成

### 4. セッションブロック生成

セッション番号は `$ARGUMENTS` から取得。未指定なら既存ノートのセッション数+1。

```markdown
## ✅ セッション{N}（{開始時刻}〜{現在時刻}）— {主な作業の要約}

| コミット | 内容 |
|---------|------|
| {hash7桁} | {コミットメッセージ} |
| {hash7桁} | {コミットメッセージ} |
...

**変更統計:** {N}ファイル変更、+{追加行} -{削除行}

### 主な成果
- {成果1}
- {成果2}
- {成果3}
```

### 5. デイリーノートに追記
- 既存ノートの `## ✅ 今日やったこと` セクションの後、または最後のセッションブロックの後に追記
- 既存セッションブロックは絶対に上書きしない
- `[x] MuscleMap開発（コミット{合計N}本）` の数字を更新

### 6. 出力
- 追記した内容のサマリーをユーザーに表示
- ファイルパスを伝える

## 注意
- コミットメッセージは原文維持（日本語/英語混在OK）
- 開始時刻が不明な場合は最初のコミット時刻を使う
- テーブル形式はデイリーノート 2026-03-21.md のスタイルに合わせる
- ファイルパス: `/Users/og3939397/Documents/Obsidian Vault/zDOG/Daily/`

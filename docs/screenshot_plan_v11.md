## v1.1 新構成（6枚）

### 設計原則

1. **1枚目で「何のアプリか」を0.3秒で伝える** — ヒーローショット
2. **2枚目で「他にない機能」を見せる** — GIF動画プレビュー（最大差別化）
3. **3-4枚目で「使い続ける理由」を示す** — ルーティン + PR達成感
4. **5枚目で「Pro機能の価値」を示す** — Strength Map + 比較表
5. **6枚目で「エコシステム」を見せる** — シェア + Watch + カレンダー
6. **各スクショは色味を変えて単調さを防ぐ**（グリーン→シアン→グリーン→ゴールド→シアン→グリーン）

---

### Shot 1: ヒーロー — 回復マップ（ファーストインプレッション）

**最も重要。App Store検索結果のサムネイルにもなる。**

| 項目 | 内容 |
|------|------|
| **日本語コピー** | 鍛えた筋肉が、**光る**。 |
| **英語コピー** | Your muscles **light up**. |
| **サブコピー** | 21部位 × リアルタイム回復マップ |
| **英語サブ** | 21 muscles × real-time recovery map |
| **画面** | ホーム画面（回復マップ前面+背面、赤/黄/緑の筋肉が光っている状態） |
| **アクセント** | グリーン (#00FFB3) |
| **装飾** | stat-chip: 「21筋肉」「92種目」「EMGベース」 |
| **強調ポイント** | 筋肉マップの色彩コントラストを最大化。「見える」体験を一目で伝える |
| **変更点 (v1.0→v1.1)** | v1.0を維持。ファーストインプレッションとして完成度が高いため |

**理由:** 回復マップはアプリの核心機能。「筋トレアプリ」としてユニークな第一印象を与える。

---

### Shot 2: GIF動画プレビュー — 「やり方が見える」（差別化）

**v1.1最大の差別化ポイント。競合にない視覚要素。**

| 項目 | 内容 |
|------|------|
| **日本語コピー** | フォームが**動画**でわかる。 |
| **英語コピー** | See the **motion**, not just the name. |
| **サブコピー** | 92種目すべてにアニメーションGIF |
| **英語サブ** | Animated GIFs for all 92 exercises |
| **画面** | ワークアウト画面のMuscleExercisePickerSheet（サムネイルGIF付きリスト） + 種目詳細の大GIF表示をオーバーレイ |
| **アクセント** | シアン (#00D4FF) |
| **装飾** | badge: 「92種目 全GIF対応」 |
| **強調ポイント** | GIFのフレームが実際に動いて見えるよう、複数フレームを並べてモーション感を演出。テキストだけの種目リストとの差を視覚化 |
| **変更点 (v1.0→v1.1)** | 新規。v1.0のshot2（紫警告）を置換 |

**理由:** GIF動画は他の筋トレアプリとの最大差別化要素。shot2の紫警告は使用頻度が低くCVR貢献度が低い。

### Shot 3: 今日のルーティン — 「迷わない」（継続価値）

**v1.1の目玉機能。ホーム画面のルーティン統合 + Day切替。**

| 項目 | 内容 |
|------|------|
| **日本語コピー** | 今日やること、**迷わない**。 |
| **英語コピー** | Never wonder **what to train**. |
| **サブコピー** | 目標×頻度×場所 → あなた専用Day分割 |
| **英語サブ** | Goals × frequency × location → your split |
| **画面** | ホーム画面の「今日のルーティン」セクション（Day 1〜5タブ表示、GIFカード2-3枚見えている状態） |
| **アクセント** | グリーン (#00FFB3) |
| **装飾** | feature-chips: 「目標」「頻度」「場所」「経験」 |
| **強調ポイント** | Day切替タブが見えること。GIFサムネイルが並んでいること。「今日のルーティン 胸」のようにDay名が表示されていること |
| **変更点 (v1.0→v1.1)** | v1.0のshot4（ルーティンビルダー）を「使っている状態」に進化 |

**理由:** オンボーディングのビルダーより「日常的に使っている画面」の方がユーザーは自分ごと化しやすい。「今日何をやるか迷わない」は最大の継続理由。

| **日本語コピー** | 自己ベスト、**自動で**検出。 |
| **英語コピー** | Auto **PR detection** & celebration. |
| **サブコピー** | PR更新をリアルタイムで祝福 + シェアカード |
| **英語サブ** | Real-time PR celebration + share card |
| **画面** | ワークアウト完了画面（PR祝福セクション + 筋肉マップハイライト + シェアカードプレビュー） |

**理由:** PR達成はユーザーの感情的ハイライト。「このアプリで鍛えたら自己ベストを祝ってもらえる」というUX的価値が高い。

### Shot 5: Strength Map + 無料 vs Pro 比較（CVR直結）

**Pro課金のCVRに直結するショット。比較表でFOMOを煽る。**

| 項目 | 内容 |
|------|------|
| **日本語コピー** | 筋力が**太さ**で見える。 |
| **英語コピー** | See your **strength** in thickness. |
| **サブコピー** | S〜Dグレードで全身を評価 |
| **英語サブ** | S-to-D grading across your body |
| **画面** | 左: Strength Map画面（筋肉の太さが異なる前面+背面）、右下: 無料 vs Pro 比較表（ペイウォールのfeatureListSection部分をスクショ切り抜き） |
| **アクセント** | シアン (#00D4FF) + グリーン (#00FFB3) グラデーション |
| **装飾** | pro-badge + grade-row (S, A, B, C, D) |
| **強調ポイント** | Strength Mapの視覚的インパクト + 比較表で「無料でも使えるがProだともっとすごい」を明確化。無料ユーザーへの「もったいない」感 |
| **変更点 (v1.0→v1.1)** | v1.0のshot5を拡張。比較表を追加して「Proの価値」を明示 |

**理由:** App Storeでのスクショは「課金前の営業資料」。比較表を見せることで無料DL後の課金CVRも上がる。

**「続けたくなる仕組み」を見せるクロージングショット。**

| 項目 | 内容 |

| **画面** | 3画面コンポジット: 左=カレンダー（履歴画面、色分けされた日付）、中央=シェアカード（ワークアウト完了のSNSカード）、右=Apple Watch画面 |
| **アクセント** | グリーン (#00FFB3) |

| **強調ポイント** | カレンダーの「色が塗られていく」視覚的達成感。シェアカードのデザイン品質。Watchの存在感 |

**理由:** 最後のスクショは「このアプリを使い続けたらどうなるか」のビジョン。カレンダーの色が埋まっていく視覚は強力な継続モチベーション。

## 2. 各競合の詳細分析

### 2-1. Hevy（最大の直接競合）

**ポジショニング:** 「世界で最も直感的なワークアウトトラッカー」

**強み:**
- 10M+ユーザー、Apple Featured
- ソーシャル機能（友達のワークアウト共有、フォロー）
- 筋肉分布ヒートマップ（ログ中に表示）
- モダンUI、寛大な無料枠
- Hevy Trainer（AI プログラム生成、2026年新機能）
- AirPods Pro 3 バイオメトリクス連携

**弱み:**
- 回復管理は表面的（ヒートマップのみ、超回復計算なし）
- プログラミング機能は後発
- カレンダー・統計の改善要望あり

**スクショ構成パターン:**
1. ワークアウトログ画面（セット入力UI）
2. 進捗グラフ・チャート
3. ルーティンビルダー
4. ソーシャルフィード
5. Apple Watch連携

## 3. スクリーンショット構成パターン分析

### 3-1. 上位アプリの共通パターン

| 順番 | 表示内容 | 採用率 |
|:---|:---|:---|
| **1枚目** | ヒーローショット（メイン画面 + キャッチコピー） | 100% |
| **2枚目** | ワークアウトログ/入力UI | 90% |
| **3枚目** | 進捗グラフ/統計 | 85% |
| **4枚目** | ルーティン/プラン管理 | 70% |
| **5枚目** | ソーシャル or Apple Watch | 60% |

### 3-2. 高CVRスクショの設計原則（2026年ベストプラクティス）

1. **最初の3枚で勝負**: ユーザーの90%は3枚目以降スクロールしない
2. **成果 > 機能**: 「何ができるか」ではなく「何が達成できるか」を訴求
3. **テキストオーバーレイ必須**: 画面キャプチャだけでは伝わらない。リボン/バナーで明示
4. **デバイスモックアップ**: 実機画像を入れた方がCVR向上
5. **一貫したブランドカラー**: 全スクショで同じカラーパレット・タイポグラフィ
6. **物語構造**: 1枚目=問題提起/価値提示 → 2枚目=解決策 → 3枚目=成果

### 3-3. MuscleMapが真似すべきパターン

| パターン | 参考アプリ | MuscleMapでの適用 |
|:---|:---|:---|
| **筋肉マップをヒーローに** | Fitbod（回復インジケーター） | 1枚目: 回復マップの全身表示 + 「筋肉の状態が見える」キャッチ |
| **ビフォーアフター感** | Strong（進捗チャート） | 3枚目: Strength Mapの成長変化 |
| **Apple Watch連携** | Hevy, Strong | 5枚目: Watch画面を含める |
| **ダーク×グリーンの世界観** | なし（独自） | 全スクショでmmAccentPrimary統一 |
| **GIF的動きの表現** | なし（独自） | 2枚目: 種目GIF + 記録UIの組み合わせ |

## 4. 差別化キーワード分析

### 4-1. 現在の競合が使っているキーワード

| キーワード | 使用アプリ | 競争度 |
|:---|:---|:---|
| 筋トレ 記録 | 筋トレMemo, バーンフィット, 他多数 | **極高** |
| workout tracker | Hevy, Strong, Jefit, 他多数 | **極高** |
| gym log | Hevy, Strong | **高** |
| 筋トレ 管理 | 筋トレMemo | **高** |
| トレーニング 分析 | バーンフィット | **中** |
| AI workout | Fitbod | **中** |

### 4-2. MuscleMapが狙うべき差別化キーワード

#### Tier 1: 独自ポジション（競合ゼロ〜極少）

| キーワード | 理由 | 推定競争度 |
|:---|:---|:---|
| **筋肉 回復 マップ** | 回復状態を2D筋肉マップで可視化するアプリは存在しない | **極低** |
| **超回復 アプリ** | 超回復を明示的に管理するアプリは皆無 | **極低** |
| **muscle recovery map** | 英語圏でも筋肉マップ×回復の組み合わせは未開拓 | **極低** |
| **recovery tracker** | Fitbodが近いが「tracker」としては名乗っていない | **低** |
| **筋肉 可視化** | 3Dモデル系はあるが回復状態の可視化は独自 | **低** |

#### Tier 2: 差別化可能（競合少〜中）

| キーワード | 理由 | 推定競争度 |
|:---|:---|:---|
| **筋トレ メニュー 提案** | 自動提案機能を持つ国内アプリは少ない | **低〜中** |
| **筋力 レベル 可視化** | Strength Map相当の機能は競合になし | **低** |
| **トレーニング 分割** | ルーティン自動生成は差別化要素 | **中** |
| **workout planner recovery** | 回復を加味したプランナーは希少 | **低** |

#### Tier 3: 補助キーワード（ロングテール）

| キーワード | 理由 |
|:---|:---|
| 筋トレ 何日 休む | 検索意図が超回復と直結 |
| 筋肉 部位 回復 時間 | MuscleMapのコア機能に直結 |
| トレーニング オーバーワーク 防止 | 回復マップの存在意義そのもの |
| muscle group recovery time | 英語圏ロングテール |
| which muscles to train today | おすすめメニュー機能に直結 |

**現在:**

## 7. 結論: MuscleMapの競争優位

### 唯一無二の差別化ポイント

1. **筋肉回復マップ**: 21筋肉の回復状態をリアルタイムで2D可視化。Fitbodがテキスト/数値で回復を表示するが、視覚的な筋肉マップは競合ゼロ
2. **超回復計算エンジン**: ボリューム係数付きの回復時間計算。セット数で回復時間が変動する本格的な計算
3. **Strength Map**: PRデータから筋肉の発達レベルを太さで可視化。類似機能を持つ競合なし
4. **未刺激警告**: 7日以上未刺激の筋肉を紫で点滅。オーバートレーニング防止だけでなく、サボり防止も
5. **回復ベースのメニュー提案**: 回復データからルールベースで「今日やるべき」メニューを自動生成

### 市場ポジション

```
        記録特化 ←―――――――――――――→ AI自動生成
           |                            |
    Strong |                     Fitbod |
    筋トレMemo                          |
           |      Hevy                  |
           |                            |
           |     ★ MuscleMap ★          |
           |   （回復可視化 × 提案）     |
           |                            |
        シンプル ←―――――――――――――→ 高機能
```

MuscleMapは「回復の可視化」という独自軸を持ち、記録ツール（Strong/Hevy）とAIプランナー（Fitbod）の中間に位置する。**「見てわかる」体験**が最大の差別化要因。

### ASO（App Store最適化）アクション

1. **タイトルに「回復」を維持**: 競合が使っていない独自キーワード
2. **「超回復」「可視化」をサブタイトルに**: 検索ボリュームは小さいが、CVR（コンバージョン率）が高い意図一致キーワード
3. **スクショ1枚目は筋肉マップ一択**: 競合と完全に差別化できるビジュアル
4. **「筋トレ 記録」は説明文で補完**: タイトル/サブタイトルには独自キーワードを優先

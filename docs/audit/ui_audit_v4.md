### 1. HomeView（ホーム画面）

- **ビジュアル品質: A**
- **HIG準拠: Yes**
- **競合比較: 8/10**
- **スクリーンショットパス:**
  - `docs/audit/screenshots/01_home_view_top.png` (上部)
  - `docs/audit/screenshots/01_home_view_bottom.png` (下部)

#### 良い点
- 筋肉マップが画面の主役として大きく表示されており、CLAUDE.mdの「ビジュアルファースト」原則に準拠
- 前面・背面の2D SVG人体図が並列表示され、回復状態が色で一目瞭然（赤=疲労、黄=中間、緑=回復、灰=未記録）
- カラーシステムがCLAUDE.md定義通り（mmBgPrimary=#121212, mmAccentPrimary=#00FFB3）
- 「14週連続」ストリークバッジが火の絵文字付きで動機付けに貢献
- ナビバーにbook/chartアイコンがあり、種目辞典・分析へのアクセスが明確
- 「今日のメニュー」セクションがマップ直下に配置され、3秒ルール（今日やることがすぐわかる）を満たす
- 「+2.5kg 挑戦！」のプログレッシブオーバーロード提案がmmAccentPrimaryで目立つ
- タブバーの4タブ構成はiOS標準パターン

#### 問題点
1. **[要改善] 人体図の頭部・足先が若干切れている可能性** — スクリーンショットでは頭頂部と足先が完全に表示されているように見えるが、デバイスサイズによっては切れるリスクがある。CLAUDE.mdの「人体図は頭から足先まで完全に表示」ルールへの厳密な準拠が必要
2. **[要改善] 「バーベルスク...」テキスト切れ** — 「バーベルスクワット」が「バーベルスク...」と省略表示されている。種目名が長い場合にlineLimit処理が必要
3. **[軽微] 「恥を晒す」ボタンのトーン** — 未刺激部位セクションの「恥を晒す」ボタンはユーモラスだが、App Store審査時にネガティブな印象を与えるリスクがある。「共有する」等に変更検討
4. **[軽微] 凡例(高負荷/回復初期/回復中/回復後期/ほぼ回復/未刺激)が小さい** — フォントサイズがcaption2相当で視認性がやや低い

#### 改善提案
- 種目名表示を`.lineLimit(2)`に変更し、長い名前でも途切れないようにする
- 凡例のフォントサイズを`.caption`に引き上げ、コントラストを改善
- 「恥を晒す」を「シェアする」等に変更

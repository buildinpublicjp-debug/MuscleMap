# UX フロー完全分析 — 2026-03-20

> **対象ユーザー**: 日本市場の筋トレ初心者（20-30代男性）
> **分析対象**: オンボーディング〜初回ワークアウト〜Paywallまでの全フロー
> **分析者**: Claude Code UX Audit

---

## エグゼクティブサマリー

1. **強み**: 視覚的インパクト（筋肉マップ、GIF、超回復アニメーション）は競合と差別化できる水準。目標選択からルーティン完成までの「自分専用感」演出は秀逸。
2. **課題**: FrequencySelectionPageの専門用語（プッシュ/プル/脚）、RoutineBuilderPageの情報過多、PaywallのValue Proposition不足が初心者離脱リスク。
3. **最大のボトルネック**: RoutineBuilderPage（Page 6）— 認知負荷が最も高く、「面倒くさい」感情のピーク。ここを簡略化すれば全体CVRが改善する可能性大。

## ページ別分析

### Page 0: SplashView

**ファイル**: `/Users/og3939397/MuscleMap/MuscleMap/Views/Onboarding/SplashView.swift`

**初心者の反応**:
- 筋肉マップが順次ライトアップするアニメーションは視覚的に印象的
- 「鍛えた筋肉が光る」「あなたの体の変化を、目で見る」というタグラインは初心者にも直感的に理解可能
- 1.5秒後に「始める」ボタンが表示される適切なタイミング

**良い点**:
- ビジュアルファーストで言葉より先に「何ができるか」を見せている
- アニメーションの段階的表示（タイトル→筋肉マップ→タグライン→ボタン）がストーリー性を持つ
- Haptic Feedbackが実装されている

**離脱リスク**: **Low**
- ファーストインプレッションとして最適。離脱する理由がほぼない

**認知負荷**: 低
- 読むテキストが最小限、視覚で理解できる

**改善案**:
1. 「MuscleMap」というアプリ名だけでは何のアプリかわからないユーザーもいる。サブタイトルで「筋トレ記録アプリ」と明示しても良い
2. ボタンラベル「始める」は「無料で始める」にすると心理的障壁が下がる

### Page 1: GoalSelectionPage

**ファイル**: `/Users/og3939397/MuscleMap/MuscleMap/Views/Onboarding/GoalSelectionPage.swift`

**初心者の反応**:
- 「なぜ鍛える？」という問いかけは初心者でも答えやすい
- 7つの選択肢は感情的で共感を呼ぶ言葉遣い（「デカくなりたい」「モテたい」「舐められたくない」）
- 複数選択可能で、選ぶと筋肉マップがリアルタイムでハイライト → 「自分専用感」が出る

**良い点**:
- 選択肢が具体的かつ感情的（「健康に長生き」より「デカくなりたい」が先に来ている）
- 選択時にHaptic + アニメーション + マップ更新という即時フィードバック
- 「重点部位」チップで選択した目標と筋肉の関連性を可視化

**離脱リスク**: **Low**
- 選択するだけで進める。視覚的報酬もある

**認知負荷**: 低
- 選択肢の意味は明確。筋トレ用語なし

**改善案**:
1. 「1つ以上選んでください」というヒントテキストがない。「次へ」ボタンがグレーアウトしているが、理由が明示されていない
2. 筋肉マップをタップすると`MuscleExerciseSheet`が開くが、この段階で種目を見せる必要性は低い。タップ誘導を外すか、タップ時の説明を変えるべき

### Page 2: FrequencySelectionPage

**ファイル**: `/Users/og3939397/MuscleMap/MuscleMap/Views/Onboarding/FrequencySelectionPage.swift`

**初心者の反応**:
- **「？」ポイント**: 「プッシュ/プル/脚」「Push / Pull / Legs」という専門用語が説明なしで表示される
- 超回復アニメーションは視覚的に面白いが、「何を意味しているか」がわからない可能性

**問題点**:
```swift
case .thrice:
    return isJapanese
        ? ["プッシュ", "OFF", "プル", "OFF", "脚", "OFF", "OFF"]
        : ["Push", "OFF", "Pull", "OFF", "Legs", "OFF", "OFF"]
```
- 「プッシュ」「プル」は筋トレ上級者向けの分割法用語。初心者は「何それ？」となる
- 「上半身/下半身で効率よく」はまだわかるが、「プッシュ/プル/脚の王道分割」は初心者には意味不明

**良い点**:
- 筋肉マップが選択に応じて動的にアニメーションする
- 色のレジェンド（刺激・回復中・未刺激）が表示されている
- 「医学的根拠テキスト」で信頼性を演出

**離脱リスク**: **Medium**
- 専門用語への抵抗感で「自分向けではない」と感じる可能性
- 「どれを選べばいいかわからない」という迷いが発生

**認知負荷**: 中
- 選択肢自体はシンプルだが、各選択肢の意味理解に時間がかかる

**改善案**:
1. **最優先**: 「プッシュ/プル/脚」を初心者向けに言い換える。例：「胸・肩の日 / 背中の日 / 脚の日」
2. 週2回を「初心者におすすめ」とラベリングする
3. 超回復アニメーションの意味を1行で説明（「赤→黄→回復完了。このサイクルでトレーニング」）
4. 「わからない場合は週2回がおすすめです」というフォールバック導線

### Page 4: ProfileInputPage (TrainingHistoryPage.swift)

**ファイル**: `/Users/og3939397/MuscleMap/MuscleMap/Views/Onboarding/TrainingHistoryPage.swift`

**初心者の反応**:
- トレーニング経験（初心者/半年/1年+/ベテラン）は答えやすい
- 体重入力のステッパーUIは直感的
- ニックネームは「任意」と明示されているので圧迫感なし

**問題点**:
- 3つの入力項目が1ページにまとまっているのは効率的だが、スクロールが必要になる可能性
- 体重の初期値が70kgだが、これがユーザーの体重と大きく異なる場合、調整に手間取る

**良い点**:
- kg/lb切り替えが実装されており、グローバル対応
- トレ歴選択でアイコン付きの視覚的フィードバック
- 「半年」「1年+」という曖昧な選択肢で厳密さを求めない設計

**離脱リスク**: **Medium**
- 体重入力を「面倒くさい」と感じる層がいる
- 「なぜ体重が必要なの？」という疑問に答えていない

**認知負荷**: 中
- 3項目の入力は多すぎではないが、目的が明示されていない

**改善案**:
1. 体重入力の上に「体重に応じた最適な重量を提案します」と目的を明示
2. 体重ステッパーの刻み幅を1kgから5kgに変更し、微調整は後でできると伝える
3. 初心者向けの説明を追加：「わからない場合は初心者を選んでください」

### Page 5: PRInputPage

**ファイル**: `/Users/og3939397/MuscleMap/MuscleMap/Views/Onboarding/PRInputPage.swift`

## Paywall コンバージョン Analysis

**ファイル**: `/Users/og3939397/MuscleMap/MuscleMap/Views/Paywall/PaywallView.swift`

### サンクコスト効果の評価

**強み**:
- オンボーディングで7-8ページの選択を経ており、サンクコストは高い
- `RoutineCompletionPage`からハードペイウォールとして表示される位置づけは適切
- ユーザーの目標・筋肉マップ・メニュープレビューがパーソナライズされている

**弱み**:
- サンクコストは高いが、「無料で始められる」選択肢があるため、課金への強制力は弱い
- 「無料ではじめる」を選んだユーザーへの再アプローチポイントが週1回制限のみ

### Value Propositionの評価

**問題点**:
```swift
featureCheckRow(isJapanese ? "種目・重量・セット数の自動提案" : "Auto-suggested exercises, weight & sets")
featureCheckRow(isJapanese ? "強さレベル & Strength Map" : "Strength Level & Strength Map")
featureCheckRow(isJapanese ? "レベルアップまでの距離表示" : "Track your progress to next level")
```
- 「Strength Map」「強さレベル」は初心者には意味がわからない
- 「レベルアップまでの距離表示」も抽象的
- 最も重要な「無制限ワークアウト記録」が明示されていない

**改善案**:
1. **無制限ワークアウト**: 「週1回制限なし、毎日記録OK」を最上位に
2. **自動メニュー**: 「今日何をすべきか毎日教えてくれる」

### 価格の見せ方

**現状**:

**良い点**:

- 「7日間無料で始める」がCTAラベル

### 心理的バランス（「無料で続ける」vs「Pro版」）

- 表示後もグレーの小さなテキスト

**評価**:
- ハードペイウォールとしては適切だが、ユーザーが「閉じ込められた」と感じるリスク
- 3秒の遅延は「強制感」を和らげつつサンクコストを最大化するバランス

**改善案**:
1. 「無料で続ける」表示までの間、Pro機能のデモ動画/アニメーションを表示
2. 「無料で続ける」後も、アプリ内で定期的にPro機能をプレビューさせる

### ワークアウト開始→セット記録→完了

**流れの評価**:
1. **ワークアウト開始**: `WorkoutStartView`で開始ボタン or おすすめからの自動開始
2. **種目選択**: オンボーディングで作成したルーティンから提案される場合はスムーズ
3. **セット記録**: 重量/レップ入力 → セット完了
4. **完了**: `WorkoutCompletionView`でサマリー表示

**問題点**:
- 初回ユーザーが「自分で種目を選ぶ」場合、92種目から選ぶのは大変
- 重量入力時に「どの重量にすべきか」の参照点がない（初回は前回記録なし）

**改善案**:
1. 初回は「ルーティンのDay 1を始める」を大きなCTAに
2. 重量提案をPRInputPageの入力値から計算して表示
3. 休憩タイマーの自動スタートと「2分経過、次のセットへ」通知

## 英語ユーザーの体験分析

### ローカライズ品質

**良い点**:
- ほぼすべてのUIテキストが`L10n`経由または`isJapanese`分岐で日英対応
- 目標選択肢の英語版が自然（"Get Big", "Command Respect", "Look Attractive"など）
- 体重単位のkg/lb切り替え対応

**問題点**:
```swift
case .thrice:
    return isJapanese
        ? "プッシュ/プル/脚の王道分割"
        : "Push / Pull / Legs"
```
- 「Push / Pull / Legs」は英語圏では一般的だが、日本語の「プッシュ/プル」と同じ問題が残る
- 「Classic Push/Pull/Legs split」という説明が`evidenceText`にあるが、subtitle部分では省略

### 文化的適応

**日本市場向けの考慮事項**:
- 「舐められたくない」「モテたい」という感情的な表現は日本の若年男性に刺さる
- 「健康に長生き → 家族のために」も日本的な価値観

**英語圏との違い**:
- "Command Respect"は英語圏でも通じるが、日本語の「舐められたくない」ほど直接的ではない
- "Look Attractive"は日本語の「モテたい」より弱い表現

### 改善案

1. 英語版の目標説明文をより感情的に（"Turn heads at the gym"など）
2. 英語版でも「Push/Pull/Legs」に簡単な説明を追加
3. lb単位ユーザー向けのデフォルト体重を150lbに調整

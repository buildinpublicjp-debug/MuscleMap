# ワークアウト記録フロー UXレビュー

> **レビュー日:** 2026-03-21
> **対象:** ワークアウトタブ全画面（Idle → Active → Completion）
> **比較対象:** Strong / Hevy / FitNotes
> **関連ドキュメント:** `workout_flow_analysis.md`（技術分析）

---

## 1. フロー全体像と評価サマリー

### ユーザージャーニー

```
[ホーム画面]
  ├── 「ルーティン開始」ボタン → RoutineManager.pendingStartDay セット
  └── タブバー「記録」タップ → ContentView canRecordWorkout チェック
        │
        ▼
[WorkoutStartView: アイドル画面]
  ├── 筋肉マップ（タップ → MuscleExercisePickerSheet）
  ├── おすすめメニューバナー（タップ → 全種目一括追加）
  ├── お気に入り種目セクション（タップ → 種目選択）
  └── 「種目を追加して始める」→ ExercisePickerView（シート）
        │
        ▼ 種目選択でセッション自動開始
[ActiveWorkoutView: アクティブ画面]
  ├── ルーティンモード: RoutineProgressBar（完了率表示）
  ├── SetInputCard
  │   ├── GIFアニメ + タイマーバッジ + PRバッジ
  │   ├── 前回記録 +「同じ」コピーボタン
  │   ├── +2.5kg提案チップ
  │   ├── 重量ステッパー（±0.25 / 長押し±2.5）+ 直接入力
  │   ├── レップステッパー（±1）
  │   └── 「記録する」→ recordSet() → レストタイマー開始
  ├── RecordedSetsView（種目別グループ、スワイプ編集/削除）
  ├── 「種目を追加」→ ExercisePickerView
  └── 「ワークアウト終了」→ 確認ダイアログ → endSession()
        │
        ▼ fullScreenCover
[WorkoutCompletionView: 完了画面]
  ├── 紙吹雪アイコン + 目標連動コピー
  ├── 統計カード（ボリューム/種目数/セット数/時間）
  ├── シェアボタン（Instagram / システム共有）
  ├── レベルアップ祝福（PR更新+レベル変動時）
  ├── 刺激した筋肉マップ（前面+背面）
  ├── 次回おすすめ日（回復予測ベース）
  ├── Strength Mapシェア導線（PR更新時、Pro限定）
  ├── 実施種目リスト
  ├── Proバナー（非Premium時）
  └── 「閉じる」ボタン
```

### 総合評価

| 観点 | 評価 | 補足 |
|:---|:---|:---|
| 基本記録フロー | A | 種目選択→重量/レップ入力→記録の流れは明快 |
| ルーティン連携 | B- | 進捗バーはあるが次種目への導線が弱い |
| 入力効率 | B | 重量入力はGood、レップ入力に長押しなし |
| タイマー体験 | C+ | 一時停止・プリセット・スキップ不可 |
| 完了画面の達成感 | A- | 目標連動コピー・筋肉マップ・シェア充実 |
| 競合比較 | B | 基本は十分、ポリッシュ面で Strong/Hevy に劣る |

## 2. 画面別UX分析

### 2.1 アイドル画面（WorkoutIdleComponents.swift）

**良い点:**
- 筋肉マップが画面の55%を占有し、ビジュアルインパクト十分
- 筋肉タップ → 種目ピッカーの導線が直感的
- おすすめメニューが回復状態ベースで算出される
- お気に入りの水平スクロールが器具・主筋肉・筋力レベル付きでリッチ

**UX課題:**

| # | 課題 | 影響度 | 競合比較 |
|:---|:---|:---|:---|
| I-1 | おすすめメニューが空の場合、フォールバックメッセージなし | 中 | Strong: 常にメニュー候補あり |
| I-2 | 「昨日と同じワークアウト」のクイックリピートボタンがない | 中 | Hevy: 「Repeat Last Workout」1タップ |
| I-3 | ルーティン未完了Dayの再開導線がない | 中 | Strong: 「Resume Day 2」表示 |
| I-4 | お気に入りセクションにカテゴリフィルタなし | 低 | — |
| I-5 | 最近使った種目セクションがない（ピッカー内のみ） | 低 | FitNotes: 最近種目をトップ表示 |

### 2.2 セット入力画面（SetInputComponents.swift + WorkoutInputHelpers.swift）

**良い点:**
- GIFアニメで「どの種目か」が視覚的に即判断可能
- 前回記録の「同じ」コピーボタンで2タップで前回再現
- +2.5kg提案チップがプログレッシブオーバーロードを促進
- 自重種目の加重トグルが柔軟（前回加重値の復元あり）
- PR達成時のCelebrationOverlay（2秒）+ Haptic(heavy)が達成感を演出

**UX課題:**

| # | 課題 | 影響度 | 競合比較 |
|:---|:---|:---|:---|
| S-1 | レップ数ステッパーに長押し非対応（±1のみ） | 高 | Strong/Hevy: 長押しで±5 |
| S-2 | 種目名がスクロールで消える（スティッキーヘッダーなし） | 高 | Strong: 種目名が常に表示 |
| S-3 | 重量0.25kg刻みがジム標準の2.5kgと乖離 | 高 | ※workout_flow_analysis.md P3で詳述 |
| S-4 | 前回記録が全セット分ではなく直前1セット分のみ表示 | 中 | Hevy: 前回全セット展開表示 |
| S-5 | +2.5kg提案が種目カテゴリ不問（アイソレーションにも+2.5kg） | 中 | Strong: 種目別推奨増加量 |
| S-6 | 推定1RM表示がない（モチベーション材料の欠如） | 中 | Hevy: 入力中に推定1RM表示 |
| S-7 | 重量直接入力時のUX: タップ→クリア→入力→確定が4ステップ | 低 | — |

### 2.3 種目ピッカー（ExercisePickerView.swift）

**良い点:**
- GIFサムネ + ミニ筋肉マップのリッチ行が競合水準以上
- 回復状態に基づく適合性バッジ（「回復中」「未刺激」等）が独自の強み
- フィルタチップの排他切替がシンプルで迷わない

**UX課題:**

| # | 課題 | 影響度 | 競合比較 |
|:---|:---|:---|:---|
| P-1 | フィルタの複合使用不可（「最近」×「胸」のAND不可） | 中 | Strong: フィルタ複合対応 |
| P-2 | ソート選択なし（常にアルファベット順） | 中 | Hevy: 使用頻度/最終使用日ソート |
| P-3 | 「筋肉グループ」フィルタなし（カテゴリと器具のみ） | 低 | — |
| P-4 | フィルタチップのスクロールヒントなし（小画面で見切れ） | 低 | — |

### 2.4 記録済みセット一覧（RecordedSetsComponents.swift）

**良い点:**
- 種目別グループ化で視認性良好
- スワイプ編集/削除がiOS標準パターン
- 完了時刻表示でセット間ペースが把握可能
- 種目名タップで即座にSetInputCardに遷移

**UX課題:**

| # | 課題 | 影響度 | 競合比較 |
|:---|:---|:---|:---|
| R-1 | スワイプ操作のディスカバラビリティが低い（初見で気づかない） | 中 | Strong: 長押しメニュー併用 |
| R-2 | PRバッジがセッション最大に全付与（新記録と区別不能） | 中 | Hevy: 「NEW PR」と「Session Best」を分離 |
| R-3 | セットの並べ替え不可 | 低 | FitNotes: ドラッグ並べ替え |
| R-4 | RPE（体感強度）入力フィールドなし | 低 | Strong: RPE 1-10入力 |
| R-5 | セットメモ（「フォーム確認」等）の入力不可 | 低 | Hevy: セット別メモ |

### 2.5 レストタイマー（WorkoutTimerComponents.swift + WorkoutViewModel）

**良い点:**
- カウントダウン → オーバータイムの色遷移（緑→黄→赤）が直感的
- バックグラウンド復帰時のDateベース再計算で正確
- コンパクトバッジがGIF上に表示され、入力を邪魔しない
- 10秒前の警告Haptic + 完了Hapticの2段階フィードバック

**UX課題:**

| # | 課題 | 影響度 | 競合比較 |
|:---|:---|:---|:---|
| T-1 | 一時停止/再開ボタンがない（停止 or 待つのみ） | 高 | Strong/Hevy/FitNotes: 全社一時停止あり |
| T-2 | セッション中にタイマー時間変更不可（設定画面に遷移必要） | 高 | Hevy: タイマー長押しでプリセット選択 |
| T-3 | 種目変更時にタイマーがリセットされない | 中 | Strong: 種目変更でタイマーリセット |
| T-4 | 「スキップ（今すぐ次セットへ）」ボタンなし | 中 | Hevy: 「Skip」ボタン常設 |
| T-5 | 種目ごとの個別タイマー設定不可 | 低 | Strong: 種目別レスト設定 |
| T-6 | レスト中に「次の種目名」表示なし | 低 | — |

### 3.2 MuscleMapの競合優位性

1. **回復状態の可視化**: 筋肉マップが回復進捗をリアルタイム表示 → 種目選択の根拠を提供（競合にない独自機能）
2. **適合性バッジ**: ExerciseCompatibilityCalculatorが回復状態から種目の推奨度を算出 → 情報に基づいた意思決定を支援
3. **目標連動コピー**: 完了画面のメッセージがユーザーのオンボーディング目標に連動 → パーソナルな達成感
4. **筋肉マップシェアカード**: 刺激した筋肉が可視化されたシェアカードは視覚的インパクトが高い

### 3.3 MuscleMapの競合劣位性

1. **入力効率**: レップ長押し未対応 + 重量0.25kg刻み + 前回全セット非表示で、セット入力に余分なタップが必要
2. **タイマー体験**: 一時停止・プリセット・スキップの3点セットが未実装は2026年時点で標準以下
3. **種目切替**: モーダルシート経由の種目変更は、スワイプ切替やタブ表示と比べて摩擦が高い
4. **データ入力の柔軟性**: RPE、セットメモ、ドロップセット等の拡張入力が未対応

## 4. フロー上のステートマネジメントリスク

| リスク | 場所 | 影響 | 深刻度 |
|:---|:---|:---|:---|
| タイマーが種目変更時にリセットされない | `WorkoutViewModel.selectExercise()` | 新種目に古いタイマーが表示される | 中 |

| スクロール位置がセット編集シート閉じ後にリセット | `ActiveWorkoutView` | ユーザーが元の位置を見失う | 低 |

## 7. 競合と差別化すべきポイント

### 「追いつく」べき機能（テーブルステークス）

Strong/Hevyがともに実装済みで、ユーザーが「あって当然」と期待する機能:

1. タイマー一時停止/再開
2. レップ数の長押し加速
3. 前回セッション全セット表示
4. 種目間のスムーズな切替

### 「差をつける」べき独自機能（既存の強み強化）

MuscleMap独自の強みをさらに磨くことで、Strong/Hevyと異なる価値軸で勝負:

1. **回復ベース種目提案の強化**: 適合性バッジに「回復率%」を追加し、定量的な根拠を提示
2. **リアルタイム筋肉マップ更新**: セット記録のたびに筋肉マップの色が変わるアニメーション（「今のセットでここが刺激された」の即時フィードバック）
3. **推定1RMの可視化**: 入力中に推定1RMをリアルタイム計算 → Strength Mapのスコア変動予測を表示
4. **完了画面の筋肉マップシェア**: Instagram/X向けの「刺激した筋肉マップ」シェアは競合にない唯一の機能。シェア率を高めるUIの最適化が重要

---

## 8. まとめ

MuscleMapのワークアウト記録フローは**基本機能が堅実に実装されており**、特に筋肉マップ連動・回復ベース提案・目標連動コピーは競合にない独自の強み。

一方で、**入力効率とタイマー体験**の2領域で2026年のフィットネスアプリ標準を下回っている。具体的には:

- レップ長押し未対応（Strong/Hevyでは標準）
- タイマー一時停止なし（全競合で実装済み）
- 前回全セット表示なし（Strong/Hevy/FitNotesで実装済み）

Phase 1の4項目（F1-1〜F1-4）を実装すれば、競合水準に到達しつつ独自の筋肉マップ連携という差別化軸を維持できる。Phase 2以降は独自機能の深化に注力し、「筋肉の状態が見える」という唯一の価値提案をワークアウト記録フロー全体に浸透させることを推奨する。
